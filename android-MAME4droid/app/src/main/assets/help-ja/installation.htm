インストール

初回起動：ROM の保存場所

MAME4droid を初めて起動すると、ROM ファイル（および artwork、サンプル、ステートセーブなど、MAME の その他のデータ）をどこに保存したいか尋ねられます。2 つの選択肢があります：

  • 内部：MAME4droid は自身のプライベートなアプリストレージ内に空のフォルダを作成し、 それを使用します。その後、PC から USB/ADB で、またはアプリのプライベートフォルダを閲覧できる ファイルマネージャーアプリで、ROM を自分でコピーします。このデータはアプリをアンインストール すると自動的に削除されます。
    非推奨：このフォルダはアプリ専用のサンドボックス内にあるため、外部の ランチャー/フロントエンドやほとんどのファイルマネージャーからはまずアクセスできません — Android のストレージセキュリティモデルは（下記の）SAF 経由で許可されたフォルダを 前提としており、アプリのプライベートストレージは対象外です。
  • 外部ストレージ：Android 標準のフォルダピッカーを使って、デバイス上の任意の場所 （SD カード、すでに ROM に使っているフォルダ、クラウド同期フォルダなど）の既存フォルダを選びます。 MAME4droid はそのフォルダの読み書き権限を Android に要求し、次回のために記憶します。このデータは アプリのアンインストール後も残り、PC なしで通常のファイルマネージャーでファイルを自由に出し入れ できます。

重要：外部ストレージでは、選んだフォルダが そのまま あなたの ROM フォルダになります — MAME4droid はその中に roms サブフォルダを 作成しません。 ZIP 圧縮した ROM ファイルを、選んだフォルダに直接入れてください。同じフォルダ内に、以下の任意の サブフォルダを自分で作成することもできます：

artwork/ ベゼル/マーキー artwork セット用
samples/ サウンドサンプル用
software/ ソフトウェアリストメディア用（8 ビットホームコンピュータなど、取り外し可能な ソフトウェアを持つ非アーケードシステムで使われるカセット/ディスクイメージ）
sta/ ステートセーブ用。ROM パスにステートを保存も有効にした場合のみ（下記参照）

それ以外のすべて（MAME 設定、ハイスコア、NVRAM、スナップショット）は、ROM がどこにあるかに関わらず、 常に MAME4droid 自身の内部データに保存されます。

内部ストレージの場合、MAME4droid は代わりに自身のプライベートなアプリストレージ内に、下記の 完全なフォルダツリー（roms/ サブフォルダを含む）を作成します。

フォルダピッカーについて（Storage Access Framework）

Android 10 以降、アプリは共有ストレージのどこでも自由に読み書きすることが許可されなくなりました — Android 自身のピッカーを通じてユーザーが明示的に許可したフォルダにのみアクセスできます （これを Storage Access Framework、略して SAF と呼びます）。上で外部ストレージを選んだとき、 または後で ROM フォルダを変更するときに開くのがそのピッカーです。直接ファイルアクセスより少し遅い ため、MAME4droid はフォルダの一覧をキャッシュして、ROM がインデックス化されれば日常使用では高速に 保たれるようにしています。追加した ROM が表示されない場合や、フォルダ関連の不具合に気づいた場合は、 オプション->設定->一般でROM フォルダの索引をキャッシュを無効にするか、 ROM フォルダを再スキャンを実行してください。

ヒント：ROM フォルダはフラットに保ち（ROM をサブフォルダに埋めない）、無関係な非圧縮 ファイルを混ぜないでください — フォルダには本当に ZIP 圧縮した MAME ROM セットだけを入れる べきです。

後で ROM の場所を変更する

初回で正しく選ぶ必要はありません。オプション->設定->一般->ROM パスを変更で、同じ 内部/外部ストレージの選択を再度実行できます（その後アプリが再起動します）。新しい外部フォルダに 切り替える場合は、先に ROM ファイルを実際にそこへ移動またはコピーするのを忘れないでください。

ステートセーブを外部ストレージの ROM と一緒に保存したい場合（最終的にアンインストールしたときに Android がそれらを削除しないように）、オプション->設定->一般->ROM パスにステートを 保存を有効にしてください。

上級：インストール全体の再配置

ROM フォルダとは別に、上級ユーザーは MAME4droid のデータフォルダ全体（ROM だけでなく、下記に記載の すべての設定/サポートフォルダ）を オプション->設定->詳細->インストールパス で特定の パスに再配置できます。これは選択ではなく手動入力で、上級用途専用です：有効だが意図しないパスを入力 すると、MAME4droid はそれを真新しい空のインストールとして扱います。特に必要でない限り、これはそのまま にしておいてください。

フォルダ構成

これは MAME4droid の内部データフォルダツリーです（内部ストレージではそのまま使用されます。 外部ストレージでは、上で説明したとおり、下記の roms/artwork/samples/software のみが 実際に選んだフォルダに存在し、そこではフォルダ名に末尾のスラッシュが付きません）。デスクトップ MAME が使うのと同じ構成なので、既存の artwork/サンプルのコレクションを再利用できます。これらのアセットの 入手先についてはリソースとアセットを、ROM の命名 規則については対応ゲームを確認してください。

artwork/ -> Artwork ディレクトリ
cfg/ -> MAME 設定ファイルディレクトリ
hi/ -> ハイスコアディレクトリ
nvram/ -> NVRAM ファイルディレクトリ
roms/ -> ROM ディレクトリ
samples/ -> サンプルディレクトリ
snap/ -> スクリーンスナップショットディレクトリ
sta/ -> ステートセーブディレクトリ
