外部コントローラー

ここで説明する内容はすべて、USB または Bluetooth で接続した物理ゲームパッド、ジョイスティック、 アーケードスティックに当てはまります。以下はすべて オプション->設定->入力->ゲームコントローラーにあります。

コントローラーが認識される 2 つの方法

物理コントローラーがセッション中に初めてキーや意味のあるスティック操作を送ると、MAME4droid は 次の順序でその扱い方を決めます：

  1. すでに自分でマッピング済み。ボタンをマッピング（下記参照）で このコントローラーのボタンを手動で割り当てていれば、そのカスタムマッピングが常に使われます — 以下の既知プロファイルに一致する場合でも同様です。カスタムマッピングは常に優先されます。
  2. 自動検出（コントローラーを自動検出、デフォルトで有効）。 MAME4droid は既知のコントローラー名（Sixaxis/PS3、Xbox、NVIDIA Shield、MOGA、OUYA、 各種 Logitech/DragonRise/ipega など）の大きな内蔵リストを持っており、Android が報告する デバイス名がこれらのいずれかに一致すれば、そのモデルの既知のボタン配置を設定不要で自動的に 適用します。これはデバイスが報告する名前だけを見るもので、別のパッドに対して手動で行った マッピングには一切触れません。
  3. 汎用コントローラー。上記のどれにも該当しなくても、Android がそれを キーボードやリモコンや任意の HID デバイスではなく、実際のゲームパッドまたはジョイスティック として報告していれば、MAME4droid はそれでも次に空いているプレイヤー枠を割り当て、標準的な デフォルト配置（標準の D-Pad/A/B/X/Y/L1/R1/L2/R2 ボタンコード）にフォールバックします。 適切な配置にするには手動でボタンをマッピングするよう求める黄色の通知が表示されます。 ゲーム用ハードウェアとして自身を識別しないデバイスは、プレイヤー枠を取得することはありません。 Android の戻る/メニューキー（Android TV では Start も）だけは、引き続き UI のショートカット として機能します。

コントローラーを自動検出をオフにすると手順 2 は完全にスキップされます：手動でマッピング しない限り、すべてのパッドは「汎用」（手順 3）になります。

プレイヤー枠：接続と切断

最大 4 台のコントローラーが同時にプレイヤー 1～4 の座席を占めることができます。この枠は設定する ものではなくハードウェアの状態です：単に、そのコントローラーがどの検出経路で認識されたかに関わらず、 最初の入力を送った時点で空いている最初の枠（0～3）が割り当てられます。つまり、機器の電源を入れたり 接続したりする順序によって、同じ物理コントローラーがセッションごとに異なるプレイヤー番号になる ことがあります — ただしボタンマッピング自体は移動しません。それはコントローラー自体の アイデンティティ（下記の永続 ID参照）に結び付いており、枠には結び付いていないからです。

コントローラーを切断すると、その枠はすぐに解放され、保持していたボタン/スティックの状態もクリア されます（何も「押しっぱなし」のままにならないように）。そして短い通知 （「コントローラーが切断されました（Pn）」）が表示されます。同じパッドを後で再接続すると、 次に空いている枠に再度座り、永続 ID のおかげで、すでに行っていたカスタムマッピングが追加の設定なしに すぐに再適用されます。

ゲームを終了して MAME4droid 自体のインターフェースに戻ると、その時点で接続されているすべての コントローラーが最初から再評価されます（自動検出に失敗した拒否履歴はクリアされ、枠は再び配分され ます）が、保存されたカスタムマッピングが忘れられることはありません。

永続 ID：なぜ重要なのか

Android は接続時に各入力デバイスへ番号を割り当てますが、その番号は不安定です：再接続、再起動、 さらには同じパッドの USB と Bluetooth の切り替えの間でも変わります。マッピングをその番号に 結び付けると、絶えず設定し直すことになるため、MAME4droid はそうしません。代わりに Android に デバイスのハードウェアディスクリプタ（接続方法ではなくデバイス自体から Android が導き出す 指紋のようなもの）を尋ね、そのディスクリプタを初めて見たときに、1000 から始まる小さな連番の 仮想 ID に変換します。それ以降、その同じ物理コントローラーは常に同じ仮想 ID を取得し、 あなたのカスタムキーマッピングが実際に結び付けられているのは、この仮想 ID であり、Android の 接続時番号ではありません。ディスクリプタ→仮想 ID の対応表はアプリの非公開の設定に保存され、 アプリの再起動やデバイスの再起動を超えて保持されます。

影響：

  • 一度マッピングしたコントローラーは、どの USB ポートでも、Bluetooth 経由でも、再起動後でも、 他のパッドとどんな順番で混在させても、常に正しく動作し続けます。
  • 本当に異なる 2 台の物理コントローラーは、たとえ全く同じモデルであっても、常に 2 つの異なる 仮想 ID（1000、1001、1002…）を取得するため、それぞれが独立したレイアウトを持てます。
  • コントローラーが欠落した、短すぎる、または文字通り「unknown」なディスクリプタを報告する場合 （一部の安価な、または実装の悪いクローン製品でよくあります）、MAME4droid はそのための安定した 指紋を作れず、代わりに Android が提供する不安定な番号にフォールバックします。そのような パッド用に行ったマッピングは、再接続や再起動の後に一致しなくなることがあります。この フォールバック番号が同じままである保証がないためです。これに遭遇した場合は、下記の マッピングプロファイルを共有を参照してください。

マッピングプロファイルを共有（デフォルトはオフ）：実際のディスクリプタに関係なく、 すべてのコントローラーに単一の仮想 ID を共有させます（ボタンをマッピング画面では ID 9999として表示されます）。同一のクローンゲームパッドを複数所有していて、すべてに 対して一度だけボタンをマッピングしたい場合や、ディスクリプタが不安定なパッドへの対処法として オンにしてください。トレードオフとして、すべてのコントローラーが同じ ID に解決されるため、 全員が全く同じボタンレイアウトを使うことになります — これがオンの間は、プレイヤー 1 と プレイヤー 2 に異なるマッピングを与えることはできません。マルチプレイヤー自体は問題なく動作し、 レイアウトだけが共有されます。

ボタンをマッピング

ボタンをマッピングを開くと、コントローラー 1からコントローラー 4までの 4 項目のリストが表示されます — これらは特定の物理デバイスではなく、4 つのプレイヤー枠です。 いずれかを選ぶと、MAME の 16 個のアーケード入力（上/下/左/右、ボタン A～H、Coin、Start、Exit、 Option）が表示され、それぞれに現在何が割り当てられているかが示されます：未マッピングか、 あるいは「ID 1000 : BUTTON_A」のようなもの（コントローラーの仮想 ID と Android の生の キー名）です。入力項目をタップすると、キャプチャ画面（「入力を待っています……」）が開き、次に 押した物理キーを取得し、どのコントローラーから送られたかを読み取り、その仮想 ID を計算し、その （コントローラー、キー）のペアをそのプレイヤー枠の入力に結び付けます。全く同じ物理コントローラーの 全く同じ物理ボタンが、他のどこかで別のアクションにすでに割り当てられていた場合、その古い割り当ては まず静かに解除されます — 1 つの物理ボタンは一度に 1 つのアーケード入力しかトリガーできません。 その入力だけの割り当てを解除するには、同じ画面のクリアを使います。

マッピングは接続順ではなく仮想 ID で識別されるため、マッピングする際に特定のコントローラーを 特定の枠に接続しておく必要はありません。要求されたときにボタンを押すだけで、それを操作させたい プレイヤー枠に割り当てられます。

デフォルトに復元

デフォルトに復元はコントローラーごとではなく、完全な工場出荷時リセットです：4 つの プレイヤー枠すべてのカスタムマッピングをデフォルトに戻し、これまでに割り当てられたすべての 仮想 ID も忘れます（ディスクリプタの対応表がクリアされ、番号は 1000 から再スタートします）。 所有するすべてのコントローラーは、次に使用されたときに新品として扱われます：自動検出と汎用枠の 割り当てが、すべてに対してゼロからやり直されます。

アナログスティック

アナログスティックの生の X/Y がどのようにアーケード入力に変換されるかを、3 つの設定が左右します：

  • モード（デジタル D-Pad / デジタルスティック / アナログスティック）： どちらのデジタルオプションでも、スティックの動きは実際のアーケードジョイスティックが MAME に 与えるのと同じように、2/4/8 方向のデジタル方向に変換されます。アナログスティック では、代わりに生のアナログ値がそのままレーシングゲームなどのアナログ入力に対応したゲームへ 送られます。
  • スティックタイプ（Auto/2 方向/4 方向/8 方向）：上記のデジタル変換が 何方向に丸められるか。Autoは現在実行中のゲーム自体に、そのジョイスティックが 何方向に対応しているかを問い合わせます。
  • アナログデッドゾーン（ゲームコントローラー配下のもの。 タッチコントローラー配下にある画面上のスティック用の同名の設定と混同しないように 注意）：使い古した安価なスティックの小さなずれを無視するために、物理スティックが中心から どれだけ動けば入力として認識されるか。

右アナログスティックとアナログトリガーは、存在する場合、それらを使うゲームやライトガン風の操作 のために、常に生のアナログデータとして（デジタル化されずに）送られます。

画面上のコントロールを隠す

物理コントローラーが有効になると（枠に座った、あるいは単にスティックを動かしただけでも）、 MAME4droid にフロントエンド UI（MAME UI でタッチコントロールを非表示）またはゲーム中 （ゲーム中にタッチコントロールを非表示）でオンスクリーンのタッチボタンを自動的に隠させる ことができます。どちらもフルスクリーンモードでのみ適用されます。これは純粋に見た目の設定であり タッチ入力自体を無効にするわけではなく、実機を使っている間、画面上のパッドを邪魔にならないよう にするだけです。

トラブルシューティングのヒント

  • 自動検出が誤認識した（名前で認識されているのにボタンが違う）：無理に 争わず、上書きしてください。ボタンをマッピングに行き、そのコントローラーに本当に 欲しいボタンを設定します — カスタムマッピングはボタン単位で常に自動検出より優先される ため、おかしいと感じるボタンだけを再マッピングすれば十分です。
  • パッドが全く検出されない（緑や黄色の通知が一切出ず、ボタンが何も反応しない）： Android 自体がそれをゲーム用ハードウェアとして報告していないためで、MAME4droid ではどうにも できません。安価な HID アダプタや「キーボードエミュレーション」モードのままのパッドで よく見られます。パッドやそのドングルに別の USB/Bluetooth/モード切替スイッチがないか確認する か、汎用の HID-ゲームパッド変換アダプタを試してください。
  • 設定したマッピングが繰り返し「消える」（パッドを抜き差ししたり、電池を 交換したり、USB と Bluetooth を切り替えたりした後）：そのコントローラーはハードウェア ディスクリプタが不安定または欠落している可能性が高く、MAME4droid は同じデバイスとして 二度認識できません。回避策としてマッピングプロファイルを共有をオンにしてください （上記の永続 ID参照）— こうすればディスクリプタの有無にかかわらず、どの パッドも常に同じ共有マッピングに解決されます。
  • 同一のパッドを 2 台以上所有していて個別にマッピングしたくない場合：同じ 対処法、マッピングプロファイルを共有です。ただし、すべてのパッドがその 1 つの共有 レイアウトを使わされることになる点は覚えておいてください。
  • 入力が意図しないプレイヤーに割り当てられた（例：パッドが P1 ではなく P2 を 操作する）：枠は設定ではなく接続順（早い者勝ち）で割り当てられます。P1、P2、P3、P4 として 座らせたい順序でコントローラーの電源を入れる、あるいは接続してください。
  • 方向やボタンが押しっぱなしに感じる、または勝手に入力される：これは通常、 静止時でもわずかな動きを報告してしまう、使い古した/緩んだスティックが原因です。 ゲームコントローラー配下のアナログデッドゾーンを 1～2 段階上げてください。
  • 完全に最初からやり直したい（パッドの挙動がおかしく、上記のどれも効果が ない場合）：デフォルトに復元はすべてのカスタムマッピングと記憶されているすべての コントローラーの識別情報を消去するため、次にどのパッドから来る入力も、すべて新品として 扱われます。
