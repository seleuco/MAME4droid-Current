外部手柄

以下内容适用于通过 USB 或蓝牙连接的物理手柄、摇杆和街机摇杆。这里描述的所有内容都位于 选项->设置->输入->游戏手柄中。

手柄被识别的两种方式

每当某个物理手柄在一次会话中首次发送按键或明显的摇杆移动时，MAME4droid 会按以下顺序决定如何 处理它：

  1. 您已经手动映射过它。如果您在映射按键（见下文）中手动绑定过该 手柄的任何按键，即使它本可以匹配下面的已知配置，也会始终使用您的自定义映射。自定义映射 始终优先。
  2. 自动检测（自动检测手柄，默认开启）。MAME4droid 内置了一份庞大的 已知手柄名称列表（如 Sixaxis/PS3、Xbox、NVIDIA Shield、MOGA、OUYA，以及多款 Logitech/DragonRise/ipega 等型号），如果 Android 报告的设备名称与其中之一匹配，就会 自动应用该型号的已知按键布局，无需任何设置。这只检查设备上报的名称，不会影响您为其他 手柄手动创建的任何映射。
  3. 通用手柄。如果以上都没有认领它，但 Android 仍将其报告为真正的手柄或 摇杆（而不是键盘、遥控器或随便什么 HID 设备），MAME4droid 仍会为其分配下一个空闲的玩家 位，并回退到一个合理的默认布局（标准的 D-Pad/A/B/X/Y/L1/R1/L2/R2 按键代码）。您会看到 一条黄色提示，要求您手动映射按键以获得正确的布局。不将自身标识为游戏硬件的设备永远不会 占用玩家位；只有它的 Android 返回/菜单键（在 Android TV 上还有 Start）仍可作为界面 快捷键使用。

关闭自动检测手柄会完全跳过第 2 步：除非您手动映射，否则每个手柄都会成为"通用"手柄 （第 3 步）。

玩家位：连接与断开

最多可以同时有 4 个手柄占据玩家 1 至 4 的位置。这个"位"是硬件状态，不是您配置的东西：它就是 手柄第一次发送输入时，第一个空闲的位（0 到 3），无论是哪种检测方式认领了它。这意味着同一个 物理手柄在不同会话中可能落到不同的玩家编号上，这取决于设备开启或连接的先后顺序 — 不过按键映射本身不会跟着变，因为它绑定的是手柄本身的身份（见下文持久 ID），而不是 这个位。

断开手柄会立即释放它的位，清除它当时保持的任何按键/摇杆状态（这样不会有输入"卡住"），并显示 一条简短提示（"手柄已断开（Pn）"）。之后重新连接同一个手柄，会让它重新落座到下一个 空闲的位，并且由于持久 ID 的存在，会立即重新应用您之前为它做过的任何自定义映射，无需重新 设置。

退出游戏返回 MAME4droid 自身界面时，当时连接的所有手柄都会从头重新评估（自动检测失败留下的 屏蔽记录会被清除，位会重新分配），但绝不会忘记您保存的自定义映射。

持久 ID：为什么重要

Android 在每个输入设备连接时都会分配一个编号，但这个编号是不稳定的：重新连接、重启，甚至 同一个手柄在 USB 和蓝牙模式之间切换时都会变化。如果把映射绑定到这个编号上，就意味着要不断 重新设置，所以 MAME4droid 没有这样做：它转而向 Android 请求该设备的硬件描述符 （这是 Android 从设备本身而非连接方式得出的一种"指纹"），并在第一次看到该描述符时，将其 转换为一个从 1000 开始的小顺序虚拟 ID。从那以后，同一个物理手柄总会得到同一个 虚拟 ID，而您的自定义按键映射实际绑定的正是这个虚拟 ID — 而不是 Android 的连接时 编号。描述符→虚拟 ID 的对照表保存在应用的私有偏好设置中，应用重启和设备重启后依然 保留。

这意味着：

  • 只需映射一次手柄，之后无论插在哪个 USB 口、通过蓝牙连接、重启之后，还是与其他手柄以 任何顺序混用，它都会一直正常工作。
  • 两个确实不同的物理手柄，即使是完全相同的型号，也总会得到两个不同的虚拟 ID （1000、1001、1002……），因此每个都可以有自己独立的布局。
  • 如果某个手柄报告的描述符缺失、过短，或者字面上就是"unknown"（一些廉价或实现不佳的 山寨产品会这样），MAME4droid 就无法为它生成稳定的指纹，只能退回使用 Android 提供的 不稳定编号。为这样的手柄所做的映射，可能在重新连接或重启后失效，因为这个备用编号并不 保证保持不变。如果遇到这种情况，请参见下文的共享映射配置。

共享映射配置（默认关闭）：无论实际描述符是什么，强制所有手柄共享同一个 虚拟 ID（在映射按键界面中显示为ID 9999）。如果您拥有多个完全相同的山寨 手柄，只想统一映射一次按键，或者作为描述符不稳定手柄的变通方案，可以开启此选项。代价是： 由于所有手柄现在都解析为同一个 ID，它们会使用完全相同的按键布局 — 开启此选项期间， 无法为玩家 1 和玩家 2 设置不同的映射。多人游戏本身仍可正常进行，只是布局是共享的。

映射按键

映射按键会打开一个包含 4 个条目的列表，从手柄 1到手柄 4 — 这些是 4 个玩家位，而不是具体的物理设备。选择其中一个会显示 MAME 的 16 个街机 输入（上/下/左/右、按键 A 到 H、Coin、Start、Exit、Option），并为每一项显示当前绑定的内容： 要么是未映射，要么是类似"ID 1000 : BUTTON_A"这样的内容（手柄的虚拟 ID 和 Android 原始按键名称）。点按某个输入项会打开一个捕获界面（"正在等待输入……"），它会捕获 您接下来按下的物理按键，读取是哪个手柄发送的，计算其虚拟 ID，然后将该（手柄、按键）组合 绑定到这个玩家位的该输入上。如果同一个物理手柄上的同一个物理按键此前已经绑定到别处的另一个 动作，旧的绑定会先被静默清除 — 一个物理按键一次只能触发一个街机输入。使用同一界面中的 清除可以只取消该输入的映射。

由于映射是按虚拟 ID 而非连接顺序来识别的，映射时不需要把特定手柄插在特定的位上：只需在提示 时按下它的按键，即可将其分配给您想要它控制的那个玩家位。

恢复默认值

恢复默认值是一次彻底的出厂重置，而不是针对单个手柄的重置：它会将全部 4 个玩家位的 自定义映射全部还原为默认值，并且会忘记迄今为止分配过的所有虚拟 ID（描述符对照表被清空， 编号从 1000 重新开始）。您拥有的每个手柄在下次使用时都会被当作全新设备对待：自动检测和 通用槽位分配会对所有手柄从头重新进行。

模拟摇杆

有三个设置决定了模拟摇杆的原始 X/Y 值如何转换为街机输入：

  • 模式（数字 D-Pad / 数字摇杆 / 模拟摇杆）：无论选择哪个数字选项，摇杆 移动都会被转换为 2/4/8 方向的数字方向输入，效果就像真正的街机摇杆提供给 MAME 的一样。 选择模拟摇杆时，原始模拟值会被直接传递，供支持模拟输入的游戏使用（例如 赛车游戏）。
  • 摇杆类型（自动/2 方向/4 方向/8 方向）：决定上面的数字转换会归并到几个 方向。自动会向当前运行的游戏本身询问其摇杆支持多少个方向。
  • 模拟死区（位于游戏手柄下，请勿与触摸手柄下同名的设置 混淆，后者是给屏幕上的虚拟摇杆用的）：物理摇杆需要偏离中心多远才会被识别为输入，用来 忽略老化或廉价摇杆的轻微漂移。

如果存在右侧模拟摇杆和模拟扳机，它们会始终以原始模拟数据的形式传递（不做数字化处理），供 使用它们的游戏和光线枪式控制使用。

隐藏屏幕控件

一旦任何物理手柄处于激活状态（已落座到某个位，或只是移动了摇杆），您就可以让 MAME4droid 自动隐藏屏幕上的触摸按键，无论是在前端界面中（在 MAME 界面隐藏触摸控制）还是在 实际游戏过程中（游戏中隐藏触摸控制）；两者都仅在全屏模式下生效。这只是视觉上的 效果，并不会禁用触摸输入本身，只是在您使用真实手柄时把屏幕上的虚拟手柄挪开而已。

故障排查提示

  • 自动检测识别错误（按名称能识别，但按键不对）：不要与它较劲，直接 覆盖即可。前往映射按键，为该手柄设置您真正想要的按键 — 自定义映射按按键 逐一优先于自动检测，所以您只需要重新映射感觉不对的那几个按键。
  • 手柄完全没有被检测到（绿色或黄色提示始终不出现，按键毫无反应）：这是 Android 本身没有把它报告为游戏硬件，MAME4droid 对此无能为力。廉价 HID 适配器或卡在 "键盘模拟"模式的手柄常见此问题。检查手柄或其接收器上是否有不同的 USB/蓝牙/模式切换 开关，或尝试使用通用的 HID 转手柄适配器。
  • 您做的映射反复"消失"（在拔插手柄、更换电池，或在 USB 和蓝牙之间切换 之后）：该手柄很可能硬件描述符不稳定或缺失，导致 MAME4droid 无法两次将其识别为同一 设备。开启共享映射配置作为变通方案（见上文持久 ID）— 这样无论 有没有描述符，每个手柄都会始终解析为同一个共享映射。
  • 您拥有两个或更多相同的手柄，不想逐一单独映射：同样的解决办法， 共享映射配置。只是要记住，这也意味着所有手柄都被强制使用那一份共享布局。
  • 输入被分配给了错误的玩家（例如您的手柄控制的是 P2 而不是 P1）：位是 按连接顺序分配的，先到先得，与任何设置无关。请按照您希望它们就座为 P1、P2、P3、P4 的 顺序开启或连接手柄。
  • 某个方向或按键感觉卡住，或自己触发：这通常意味着摇杆老化或松动，即使 静止时也会报告微小移动。请将游戏手柄下的模拟死区调高一到两档。
  • 想完全从头开始（某个手柄行为异常，以上方法都无效）：恢复默认值 会清除所有自定义映射和所有已记住的手柄身份，这样下次任何手柄发来的输入都会被当作全新 设备处理。
