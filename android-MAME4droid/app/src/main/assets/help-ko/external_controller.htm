외부 컨트롤러

여기서 설명하는 모든 내용은 USB나 Bluetooth로 연결한 물리적 게임패드, 조이스틱, 아케이드 스틱에 적용됩니다. 아래 내용은 모두 옵션->설정->입력->게임 컨트롤러에 있습니다.

컨트롤러를 인식하는 두 가지 방법

물리 컨트롤러가 세션 중 처음으로 키 입력이나 의미 있는 스틱 움직임을 보내면, MAME4droid는 다음 순서로 이를 처리할 방법을 결정합니다:

  1. 이미 직접 매핑한 경우. 버튼 매핑(아래 참조)에서 이 컨트롤러의 버튼을 손으로 직접 지정했다면, 아래의 알려진 프로필과 일치하더라도 그 사용자 지정 매핑이 항상 사용됩니다. 사용자 지정 매핑이 항상 우선합니다.
  2. 자동 감지(컨트롤러 자동 감지, 기본적으로 켜짐). MAME4droid는 알려진 컨트롤러 이름의 방대한 내장 목록(Sixaxis/PS3, Xbox, NVIDIA Shield, MOGA, OUYA, 여러 Logitech/DragonRise/ipega 등 모델)을 가지고 있으며, Android가 보고하는 기기 이름이 이 중 하나와 일치하면 별도 설정 없이 해당 모델의 알려진 버튼 배치를 자동으로 적용합니다. 이는 기기가 보고하는 이름만 확인할 뿐, 다른 패드에 대해 직접 만든 매핑은 전혀 건드리지 않습니다.
  3. 일반 컨트롤러. 위 두 가지 중 어느 것도 해당하지 않지만 Android가 여전히 이를 키보드, 리모컨, 임의의 HID 기기가 아니라 실제 게임패드나 조이스틱으로 보고한다면, MAME4droid는 그래도 다음으로 비어 있는 플레이어 슬롯을 할당하고 합리적인 기본 배치 (표준 D-Pad/A/B/X/Y/L1/R1/L2/R2 버튼 코드)로 대체합니다. 올바른 배치를 위해 버튼을 직접 매핑하라는 노란색 알림이 표시됩니다. 게임용 하드웨어로 스스로를 식별하지 않는 기기는 플레이어 슬롯을 차지하지 못합니다. Android의 뒤로/메뉴 키(Android TV에서는 Start도)만 UI 단축키로 계속 작동합니다.

컨트롤러 자동 감지를 끄면 2단계를 완전히 건너뜁니다: 직접 매핑하지 않는 한 모든 패드가 "일반"(3단계)이 됩니다.

플레이어 슬롯: 연결과 연결 해제

최대 4대의 컨트롤러가 동시에 플레이어 1~4 자리를 차지할 수 있습니다. 이 슬롯은 설정하는 것이 아니라 하드웨어 상태입니다: 어떤 감지 경로로 인식되었든, 컨트롤러가 첫 입력을 보내는 순간 비어 있는 첫 번째 슬롯(0~3)이 그냥 배정됩니다. 즉, 기기를 켜거나 연결한 순서에 따라 같은 물리 컨트롤러가 세션마다 다른 플레이어 번호를 받을 수 있습니다 — 다만 버튼 매핑 자체는 바뀌지 않는데, 이는 슬롯이 아니라 컨트롤러 자체의 정체성(아래 영구 ID 참조)에 연결되어 있기 때문입니다.

컨트롤러 연결을 끊으면 즉시 슬롯이 해제되고, 가지고 있던 버튼/스틱 상태가 지워지며(아무것도 "눌린 채로" 남지 않도록), 짧은 알림("컨트롤러 연결 끊김(Pn)")이 표시됩니다. 나중에 같은 패드를 다시 연결하면 다음으로 비어 있는 슬롯에 다시 자리 잡고, 영구 ID 덕분에 이미 만들어 둔 사용자 지정 매핑이 추가 설정 없이 즉시 다시 적용됩니다.

게임을 종료하고 MAME4droid 자체 인터페이스로 돌아오면 그 시점에 연결된 모든 컨트롤러가 처음부터 다시 평가되지만(자동 감지 실패로 인한 차단 기록은 지워지고 슬롯이 다시 배정됨), 저장된 사용자 지정 매핑은 절대 잊히지 않습니다.

영구 ID: 왜 중요한가

Android는 연결될 때마다 각 입력 기기에 번호를 부여하지만, 이 번호는 불안정합니다: 재연결, 재부팅, 심지어 같은 패드의 USB와 Bluetooth 모드 사이에서도 바뀝니다. 매핑을 이 번호에 묶으면 계속 다시 설정해야 하므로, MAME4droid는 그렇게 하지 않습니다. 대신 Android에 기기의 하드웨어 디스크립터(연결 방식이 아니라 기기 자체에서 Android가 얻어내는 일종의 지문)를 요청하고, 그 디스크립터를 처음 본 순간 1000부터 시작하는 작은 순차 가상 ID로 변환합니다. 그 이후로는 같은 물리 컨트롤러가 항상 같은 가상 ID를 받으며, 여러분의 사용자 지정 키 매핑이 실제로 연결되어 있는 것은 Android의 연결 시점 번호가 아니라 바로 이 가상 ID입니다. 디스크립터→가상 ID 표는 앱의 비공개 설정에 저장되며 앱 재시작과 기기 재부팅 후에도 유지됩니다.

의미하는 바:

  • 컨트롤러를 한 번 매핑하면, 어떤 USB 포트에서든, Bluetooth를 통해서든, 재부팅 후에도, 다른 패드와 어떤 순서로 섞여도 계속 올바르게 작동합니다.
  • 정말로 서로 다른 두 물리 컨트롤러는 완전히 같은 모델이라도 항상 서로 다른 가상 ID (1000, 1001, 1002...)를 받으므로, 각각 독립적인 배치를 가질 수 있습니다.
  • 컨트롤러가 없거나, 너무 짧거나, 말 그대로 "unknown"인 디스크립터를 보고하면(일부 저가형 또는 구현이 부실한 복제품이 이렇습니다), MAME4droid는 안정적인 지문을 만들 수 없어 Android가 제공하는 불안정한 번호로 대체합니다. 이런 패드용으로 만든 매핑은 재연결이나 재부팅 후 더 이상 일치하지 않을 수 있는데, 이 대체 번호가 그대로 유지된다는 보장이 없기 때문입니다. 이런 문제를 겪는다면 아래의 매핑 프로필 공유를 참고하세요.

매핑 프로필 공유(기본적으로 꺼짐): 실제 디스크립터와 상관없이 모든 컨트롤러가 하나의 가상 ID를 공유하도록 강제합니다(버튼 매핑 화면에는 ID 9999로 표시됨). 동일한 복제 게임패드를 여러 개 가지고 있어서 모두에 대해 버튼을 한 번만 매핑하고 싶을 때, 또는 디스크립터가 불안정한 패드에 대한 해결책으로 켜세요. 단점은 이제 모든 컨트롤러가 같은 ID로 처리되므로 모두 정확히 같은 버튼 배치를 사용하게 된다는 것입니다 — 이 옵션이 켜져 있는 동안에는 플레이어 1과 플레이어 2에 서로 다른 매핑을 줄 수 없습니다. 멀티플레이어 자체는 문제없이 작동하며, 배치만 공유됩니다.

버튼 매핑

버튼 매핑을 열면 컨트롤러 1부터 컨트롤러 4까지 4개 항목의 목록이 나타납니다 — 이는 특정 물리 기기가 아니라 4개의 플레이어 슬롯입니다. 하나를 선택하면 MAME의 16개 아케이드 입력(상/하/좌/우, 버튼 A~H, Coin, Start, Exit, Option)이 표시되고, 각각에 현재 무엇이 연결되어 있는지 보여줍니다: 매핑 안 됨이거나, 또는 "ID 1000 : BUTTON_A"(컨트롤러의 가상 ID와 Android의 원시 키 이름)와 같은 형태입니다. 입력 항목을 탭하면 캡처 화면("입력을 기다리는 중...")이 열려 바로 다음에 누르는 물리 키를 잡아내고, 어느 컨트롤러가 보냈는지 읽고, 그 가상 ID를 계산한 뒤, 그 (컨트롤러, 키) 쌍을 해당 플레이어 슬롯의 입력에 연결합니다. 정확히 같은 물리 컨트롤러의 정확히 같은 물리 버튼이 다른 곳에서 이미 다른 동작에 연결되어 있었다면, 그 이전 연결은 먼저 조용히 해제됩니다 — 물리 버튼 하나는 한 번에 하나의 아케이드 입력만 일으킬 수 있습니다. 같은 화면의 지우기로 그 입력 하나만의 매핑을 해제할 수 있습니다.

매핑은 연결 순서가 아니라 가상 ID로 식별되므로, 매핑하는 동안 특정 컨트롤러를 특정 슬롯에 꽂아 둘 필요가 없습니다. 원하는 플레이어 슬롯을 위해, 요청받을 때 그 컨트롤러의 버튼을 누르기만 하면 됩니다.

기본값 복원

기본값 복원은 컨트롤러별이 아니라 완전한 초기화입니다: 4개 플레이어 슬롯 전체의 사용자 지정 매핑을 기본값으로 되돌리고, 지금까지 배정된 모든 가상 ID도 잊습니다(디스크립터 표가 지워지고 번호는 1000부터 다시 시작합니다). 가지고 있는 모든 컨트롤러는 다음에 사용될 때 완전히 새 기기로 취급됩니다: 자동 감지와 일반 슬롯 배정이 모두에 대해 처음부터 다시 실행됩니다.

아날로그 스틱

아날로그 스틱의 원시 X/Y 값이 아케이드 입력으로 변환되는 방식은 세 가지 설정이 결정합니다:

  • 모드(디지털 D-Pad / 디지털 스틱 / 아날로그 스틱): 두 디지털 옵션 중 어느 쪽이든, 스틱 움직임은 실제 아케이드 조이스틱이 MAME에 주는 것과 같은 방식으로 2/4/8방향 디지털 방향으로 변환됩니다. 아날로그 스틱에서는 대신 원시 아날로그 값이 그대로 전달되어, 레이싱 게임처럼 아날로그 입력을 지원하는 게임에 사용됩니다.
  • 스틱 유형(Auto/2방향/4방향/8방향): 위의 디지털 변환이 몇 방향으로 맞춰지는지를 정합니다. Auto는 현재 실행 중인 게임 자체에 조이스틱이 몇 방향을 지원하는지 물어봅니다.
  • 아날로그 데드존(게임 컨트롤러 아래에 있는 것이며, 화면 위 가상 스틱용인 터치 컨트롤러 아래의 동일한 이름의 설정과 혼동하지 마세요): 낡거나 저렴한 스틱의 작은 흔들림을 무시하기 위해, 물리 스틱이 중앙에서 얼마나 움직여야 입력으로 인식되는지입니다.

오른쪽 아날로그 스틱과 아날로그 트리거는 존재할 경우 이를 사용하는 게임 및 라이트건 방식 조작을 위해 항상 원시 아날로그 데이터로(디지털화되지 않고) 전달됩니다.

화면 조작부 숨기기

물리 컨트롤러가 활성화되면(슬롯에 자리 잡거나 단순히 스틱을 움직이기만 해도), MAME4droid가 프런트엔드 UI(MAME UI에서 터치 컨트롤 숨기기)나 실제 게임 플레이 중 (게임 중 터치 컨트롤 숨기기)에 화면상의 터치 버튼을 자동으로 숨기도록 할 수 있습니다. 둘 다 전체 화면 모드에서만 적용됩니다. 이는 순전히 시각적인 기능으로 터치 입력 자체를 비활성화하지 않으며, 실제 컨트롤러를 사용하는 동안 화면상의 패드를 그저 치워 줄 뿐입니다.

문제 해결 팁

  • 자동 감지가 잘못 인식한 경우(이름으로는 인식하지만 버튼이 틀린 경우): 억지로 맞서지 말고 그냥 덮어쓰세요. 버튼 매핑으로 가서 해당 컨트롤러에 실제로 원하는 버튼을 설정하면 됩니다 — 사용자 지정 매핑은 버튼 단위로 항상 자동 감지보다 우선하므로, 이상하게 느껴지는 버튼만 다시 매핑하면 됩니다.
  • 패드가 전혀 감지되지 않는 경우(녹색이나 노란색 알림이 전혀 뜨지 않고 버튼도 아무 반응이 없는 경우): Android 자체가 이를 게임용 하드웨어로 보고하지 않는 것이며, MAME4droid가 이를 바꿀 방법은 없습니다. 저가형 HID 어댑터나 "키보드 에뮬레이션" 모드에 고정된 패드에서 흔히 발생합니다. 패드나 동글에 다른 USB/Bluetooth/모드 전환 스위치가 있는지 확인하거나, 범용 HID-게임패드 변환 어댑터를 사용해 보세요.
  • 만들어 둔 매핑이 자꾸 "사라지는" 경우(패드를 뽑았다 다시 꽂거나, 건전지를 교체하거나, USB와 Bluetooth를 전환한 뒤): 그 컨트롤러는 하드웨어 디스크립터가 불안정하거나 없을 가능성이 높아, MAME4droid가 이를 같은 기기로 두 번 인식하지 못합니다. 해결책으로 매핑 프로필 공유를 켜세요(위의 영구 ID 참조) — 그러면 디스크립터 유무와 상관없이 모든 패드가 항상 같은 공유 매핑으로 처리됩니다.
  • 동일한 패드를 2개 이상 가지고 있어 각각 따로 매핑하고 싶지 않다면: 같은 해결책인 매핑 프로필 공유를 사용하세요. 다만 이는 모든 패드가 그 하나의 공유 배치를 강제로 사용하게 된다는 점을 기억하세요.
  • 엉뚱한 플레이어가 입력을 받는 경우(예: 패드가 P1이 아니라 P2를 조작): 슬롯은 어떤 설정이 아니라 연결 순서, 즉 먼저 온 순서대로 배정됩니다. P1, P2, P3, P4로 앉히고 싶은 순서대로 컨트롤러 전원을 켜거나 연결하세요.
  • 방향이나 버튼이 눌린 채로 있거나 저절로 입력되는 경우: 대개 가만히 있어도 미세한 움직임을 보고하는 낡거나 헐거운 스틱 때문입니다. 게임 컨트롤러 아래의 아날로그 데드존을 한두 단계 높이세요.
  • 완전히 처음부터 다시 시작하고 싶은 경우(패드가 이상하게 동작하고 위의 방법이 모두 소용없을 때): 기본값 복원은 모든 사용자 지정 매핑과 기억된 모든 컨트롤러 정체성을 지우므로, 그다음 어떤 패드에서 오는 입력이든 완전히 새것으로 취급됩니다.

뒤로
